## 2024-05-24 - Batching touch/wheel events with RequestAnimationFrame
**Learning:** Event listeners like `touchmove` and `wheel` can easily fire faster than the browser can paint, causing layout thrashing and main thread blocking if DOM mutations (like `style.transform`) are applied synchronously.
**Action:** Throttle visual updates (DOM mutations) to the display refresh rate using `requestAnimationFrame`. Instead of updating the DOM on every single `wheel` or `touchmove` event, update state variables, request an animation frame, and apply the final state within the RAF callback. This prevents redundant work and improves scroll performance.
